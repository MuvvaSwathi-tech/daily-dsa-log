## 📋 Problem Template

Every problem solution should include:

1. **Problem Statement** - Clear description with examples
2. **Approach** - High-level strategy and pattern
3. **Solution** - Clean, well-commented code
4. **Complexity Analysis** - Time and space breakdown
5. **Edge Cases** - Special scenarios handled
6. **Test Cases** - Verification examples
7. **Key Learnings** - Insights and common mistakes
8. **Similar Problems** - Related challenges
9. **Status Checklist** - Mastery validation

👉 See [TEMPLATE.md](TEMPLATE.md) for the full template

## 🎓 How to Use

### When Learning a Problem

1. Read problem statement
2. Try solving on your own (30-45 min)
3. Compare your approach with this solution
4. Understand differences and improvements
5. Implement from scratch without reference
6. Update progress tracking

### When Adding a Solution

1. Copy [TEMPLATE.md](TEMPLATE.md)
2. Fill in all sections thoroughly
3. Add test cases that verify the solution
4. Link from this file
5. Update [PROGRESS.md](../PROGRESS.md)

### For Quick Reference

- Check by difficulty level (medium vs hard)
- Search by pattern (two-pointers, DP, etc.)
- Look at similar problems for technique patterns

---

## 💡 Best Practices

✅ **Do:**
- Explain your thought process
- Include multiple approaches
- Test edge cases thoroughly
- Show complexity trades-offs
- Reference similar problems
- Update progress after solving

❌ **Don't:**
- Copy solutions without understanding
- Skip the complexity analysis
- Ignore edge cases
- Leave code without comments
- Treat problems as one-time events (revisit monthly)

---

## 🔄 Revision Strategy

### Weekly Revision
1. Pick 3-5 solved problems randomly
2. Solve without looking at your solution
3. Compare approach and complexity
4. Note any improvements

### Monthly Review
1. High-level: What patterns did you learn?
2. Weak areas: Which patterns need more practice?
3. Optimization: Any better approaches discovered?
4. Progress: Update metrics in PROGRESS.md
