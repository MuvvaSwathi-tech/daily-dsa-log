## 🎓 Problem-Solving Methodology

Every problem is solved with this approach:

| Stage | Focus |
|-------|-------|
| 1️⃣ **Understand** | Read carefully, ask clarifying questions |
| 2️⃣ **Plan** | Discuss approach before coding |
| 3️⃣ **Code** | Write clean, well-commented solution |
| 4️⃣ **Test** | Verify with test cases & edge cases |
| 5️⃣ **Analyze** | Explain time & space complexity |

## 🎬 Quick Start

### 1. Pick a Pattern

Choose from [PATTERNS.md](PATTERNS.md)—start with **Two Pointers** if new.

### 2. Understand the Pattern

Read the when-to-use, key signals, and template in the pattern guide.

### 3. Solve a Problem

- Check `patterns/two-pointers/` for examples
- Use `problems/TEMPLATE.md` for your walkthrough
- Document approach, solution, and complexity analysis

### 4. Track Progress

Update `PROGRESS.md` after each solve.

### 5. Review & Optimize

Revisit solutions weekly, identify weak patterns, deepen understanding.

## 💡 Problem Solving Tips

✅ **Before Coding:**
- Clarify the problem
- Discuss approach
- Identify the pattern
- Time & space trade-offs

✅ **While Coding:**
- Write clean, readable code
- Add comments for complex logic
- Test mentally with examples
- Check edge cases

✅ **After Coding:**
- Walk through test cases
- Analyze full complexity
- Discuss optimizations
- Compare with alternatives
